# Galinstan_Controller: 借助视觉反馈的液态金属控制平台

> **“将连续的物理流场离散化为数字信号，再将逻辑决策还原为物理界面的定向运动。”**
> - 本项目利用改装的 3D 打印平台与计算机视觉，构建了一个针对 NaOH 溶液中液态金属（Galinstan）电毛细效应（Electrocapillarity）的闭环控制系统。
> - 在 B 站 UP 主 [大鱼DIY] 的 3D 打印机开源项目基础上，本团队根据需求删繁就简，进行了独创。
> - 原创跳转链接：[大鱼DIY 空间](https://space.bilibili.com/488684813/upload/video)

## 2. 系统架构 (System Architecture)
系统由四个解耦的模块构成，通过 `config.json` 协议与UI界面，实现可交互的通讯：

* **[Sense] 视觉捕捉层 (`src/vision`)**：利用 OpenCV 库。引入 CLAHE 增强与双边滤波，将液滴降维映射为状态空间的坐标矩阵 $(x, y, t)$。
* **[注：]**由于传统的视觉方案容易捕捉到光线阴影而干扰识别，目前正在尝试新方案 - 通过提取液态金属的形态特征（特征提取工程），过滤环境噪音，进行精确视觉识别。
* **[Think] 动力学分析 (`src/analysis`)**：物理公式离散层面。$e(t) = x_{target} - x(t)$，利用PID 算子将位移偏差，逆运算为理想电压指令。
* **[Act] 物理执行层 (`src/control`)**：将数字逻辑映射为物理电极的 PWM 电信号，包含安全限幅与死区补偿逻辑。
* **[Bus] 共享协议层 (`config.json`)**：作为系统的“共享寄存器”，支撑视觉参数与控制参数的实时用户交互更新。



## 3. 知识资产树 (Directory Structure) 
```text
├── README.md               # 项目白皮书与架构说明
├── environment.yml         # Conda 'fluid' 运行环境镜像，以便随时复现
├── config.json             # 实时参数储存"容器"
├── /drivers                # 硬件接口封装：camera_check
├── /src                    # 核心逻辑源码
│   ├── vision/             # tracker.py: 动态追踪与滤波算法
│   ├── analysis/           # Brain.py: 电驱动力与粘性阻力演算模型
│   ├── control/            # actuator.py: 串口通信与执行器映射
│   └── ui/                 # dashboard.py: Streamlit 交互式参数寻优平台
└── /tests                  # 离线仿真与模块测试
    └── test_offline_replay.py # 数字孪生回放系统：支持实验数据回溯
```

## 4. 演进路线图 (Evolution Roadmap)
* **阶段一：离散开环（已完成）**
  * 建立 Baseline：输入坐标 $\rightarrow$ 物理建模 $\rightarrow$ 执行输出。
* **阶段二：一维闭环反馈（当前攻坚，已建立Baseline）**
  * 引入实时视觉误差补偿，建立闭环反馈系统，克服氧化层演变带来的非线性摩擦力干扰。
* **阶段三：二维矢量合成（未来资产）**
  * 对平板电极进行空间离散化，合成指向目标坐标的合力矢量 $\vec{F}_{total}$，实现复杂轨迹运动。

## 5. 部署与复现 (Deployment & Reproduction)
### 软件环境
```bash
# 创建并激活物理仿真环境
conda env create -f environment.yml
conda activate fluid
```
### 引擎启动
1. **启动控制中枢（仪表盘）**：
   ```bash
   streamlit run src/ui/dashboard.py
   ```
2. **唤醒执行引擎**：
   ```bash
   python src/main.py
   ```
* **快捷键操作**：按下 `[t]` 切换目标点，按下 `[s]` 启动 **4秒瞬态捕捉**（同步录制 Raw/Mask/Blob 数据）。

## 6. 收获与反思
#### ① 对“理论-实验-代码”三者关系的重新认识

这次项目彻底打破了我以前“理论就是公式，实验就是验证”的固有印象。

- **工程与理论的解耦**：我发现为了让系统动起来，视觉识别和串口通信这些“工程活”反而是最耗精力的。但真正决定系统“聪明程度”的，还是 `Brain.py` 里的理论模型。理论好，计算量就小，反馈就快；理论不行，代码写得再华丽，液滴也动不起来。
- **实验是代码的“定心丸”**：以前写代码总觉得参数是随手设的，现在明白代码里每一个物理常数（比如起步电压 $V_{deadzone}$）都必须从实验中实测出来。没有实验数据的支撑，代码逻辑就是空中楼阁。

#### ② 规范的软件工程“套路”
以前写程序就是一个单独的 `.py` 文件，跑通就行。这次通过学习github中的一些项目代码，我学到了什么是“正规军”的写法：

- **环境隔离**：学会了配置镜像环境文件，再也不用担心换台电脑代码就跑不起来。
- **结构化思维**：习惯了 `src` 存放源码、`README` 文件夹框架。这种规范化虽然增加了前期的工作量，但到了后期调试时，确实大大减少了找 Bug 的时间。
- **良好的保存习惯**：借助github的快照、云端同步等功能，非常便捷地编写代码日志、修复bug、迭代版本
- 
#### ③ 从“象牙塔”到“试验台”
这次课外实践让我受益匪浅，这种收获是课堂上给不了的：
- **工具链的打通**：通过 OpenCV 搞视觉、用 `os` 和 `sys` 调文件、写串口协议控硬件，我第一次完整理解了计算机是怎么指挥物理实体的。
- **心态的转变**：以前总觉得实现想法很难，现在通过一行行代码把 Galinstan 液滴“驯服”，让我觉得只要逻辑闭环、动手去试，纸上的公式真的能变成现实中的动力。这种“搞定问题”的底气，对我未来的科研信心非常重要。

#### ④ 下一步计划：进军 C++ 追求性能
Python 虽然写起来爽，但还是不够”底层“。为了应对以后更复杂的科研场景，我需要继续自学 C++，去理解更底层的内存管理和硬件响应逻辑，继续提高自己”实现想法“的能力。

#### ⑤ AI的规范使用
AI极大地降低了学习成本，以往需要不断去各大论坛、项目文件中学习别人方案，现在可以利用AI帮助检索信息，极大的节省了无意义的翻找过程。学习某个算法的时候，也可以让AI生成一个案例，帮助自己理解。
但AI是一把双刃剑，在带来便利的同时，非常容易使人依赖。在使用AI的过程中应当在心中给自己设置一个底线：“我完成这个项目一定是要学到新知识，切忌全权外包给AI，导致失去独立思考的能力”。
简单来说，就是要“知其然，且知其所以然”
